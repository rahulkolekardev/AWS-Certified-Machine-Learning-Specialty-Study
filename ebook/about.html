About this ebook

Purpose

This ebook is a concise, offline-friendly study companion for the AWS Certified Machine Learning – Specialty exam. It focuses on the AWS services and architectural patterns that most frequently appear in exam questions and real-world ML solutions.

Sources & citation

Content is paraphrased and summarized from official AWS documentation, AWS blogs, and AWS conference sessions. Each service page includes:

  • One or more links to the official AWS “What is X?” or service overview documentation pages.
  • Optionally, links to AWS blogs, workshops, or re:Invent talks for deeper dives.
  • A citation line with the primary source URL and an approximate “last updated” date.

Only short quotations (if any) are used, limited to at most 25 words and marked in blockquotes with an explicit source URL.

Canonical documentation entry points include, for example:

  • Amazon SageMaker documentation
  • Amazon Kinesis Data Streams documentation
  • Amazon S3 User Guide
  • AWS Machine Learning landing page

Due to the offline build environment, the content here is based on the latest AWS documentation known to the model at training time, not a live crawl. Always verify details against the current AWS documentation.

Updating the ebook

  1. Open the relevant service page in the ebook/services/ folder.
  2. Compare the content to the latest AWS documentation (for example, the service’s “What is X?” page and recent “What’s New” announcements).
  3. Adjust summaries, best practices, and exam tips to reflect newly released features or deprecations.
  4. Update the citation line with the latest docs URLs and the date you last reviewed them.
  5. Run ./build.sh to regenerate the ZIP archive and manifest.

Known gaps and limitations

  • Live HTML fetching from AWS documentation was not possible in the current execution environment. The summaries therefore reflect AWS documentation as of the model’s training cutoff rather than the exact state on build day.
  • Only high-level architecture diagrams are included. For complex production designs, refer to AWS reference architectures and Well-Architected Labs.
  • Code examples are intentionally minimal and omit full error handling, configuration management, and infrastructure-as-code templates.

If you notice inaccuracies or want deeper coverage (for example, feature store patterns or advanced observability for ML), extend the relevant pages and re-run the build.
