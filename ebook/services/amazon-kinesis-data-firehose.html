Amazon Kinesis Data Firehose

Amazon Kinesis Data Firehose is a fully managed service for delivering streaming data to S3, Redshift, OpenSearch Service, and HTTP endpoints. It abstracts away consumer management, buffering, and retry logic, so you can focus on specifying destinations and transformation steps.

Key features

  • Managed ingestion and delivery; no need to write consumer applications.
  • Automatic batching, compression, and encryption before delivery.
  • Optional data transformation with Lambda (lightweight) or custom processors.
  • Native integration with S3, Redshift, OpenSearch Service, and third-party endpoints.
  • Near real-time latency (typically seconds) rather than sub-second.

Where it fits in ML pipelines

  • Low-ops ingestion path from apps into S3-based data lakes for training datasets.
  • Continuous delivery of logs and metrics into OpenSearch or S3 for model monitoring.
  • Feeding Redshift with streaming data to support near real-time analytics feeding ML.
  • Preprocessing and normalizing events through Lambda before landing in storage.

How it works (high level)

CLI and SDK examples

Create an S3 delivery stream (CLI)

aws firehose create-delivery-stream \
  --delivery-stream-name ml-logs-to-s3 \
  --delivery-stream-type DirectPut \
  --s3-destination-configuration file://s3-destination.json

Put a record (Python, boto3)

import boto3, json

firehose = boto3.client("firehose")

payload = {"event": "prediction", "score": 0.82}
firehose.put_record(
    DeliveryStreamName="ml-logs-to-s3",
    Record={"Data": (json.dumps(payload) + "\n").encode("utf-8")},
)

Best practices for ML workloads

  • Use Firehose when you do not need custom streaming consumers or replay.
  • Choose appropriate buffer sizes and intervals to balance latency and S3 request costs.
  • Compress data (for example, GZIP or Parquet) to reduce storage and improve downstream query performance.
  • Use Lambda transforms to clean and partition data before landing in S3.
  • Configure error logging and backup to an S3 bucket for failed records.

Exam tips

  • Firehose is delivery focused; it cannot be used for arbitrary consumer applications.
  • Look for phrases like “no custom consumer code” and “deliver to S3/Redshift/OpenSearch”.
  • For replay or multiple independent consumers, the exam usually points to Kinesis Data Streams instead.
