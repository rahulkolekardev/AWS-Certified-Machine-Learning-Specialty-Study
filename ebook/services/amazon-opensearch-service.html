Amazon OpenSearch Service

Amazon OpenSearch Service is a managed service for deploying and operating OpenSearch and legacy Elasticsearch clusters at scale. It provides search, log analytics, observability, and vector search capabilities, which are increasingly used in ML-powered applications.

Key features

  • Managed OpenSearch clusters with automated provisioning, patching, and scaling.
  • Full-text search, aggregations, and log analytics with OpenSearch Dashboards.
  • Support for vector search and k-NN for similarity and retrieval-augmented generation (RAG) scenarios.
  • Integration with Kinesis Data Firehose and CloudWatch Logs for log ingestion.
  • Fine-grained access control and VPC support.

Where it fits in ML pipelines

  • Storing logs and metrics used to monitor model performance and behavior.
  • Indexing documents and features for search or recommendation systems.
  • Serving as a vector store for embeddings in RAG architectures with Bedrock or other LLMs.
  • Powering interactive dashboards used by data scientists and SREs.

How it works (high level)

CLI and SDK examples

Most interactions are via the OpenSearch APIs over HTTPS; the AWS CLI is mainly used for provisioning.

Describe a domain (CLI)

aws opensearch describe-domain \
  --domain-name ml-search-domain

Index a document (Python, requests)

import requests, json

endpoint = "https://ml-search-domain-xyz.region.es.amazonaws.com"
doc = {"id": "123", "title": "Prediction result", "score": 0.82}

resp = requests.put(f"{endpoint}/predictions/_doc/123", json=doc, timeout=5)
resp.raise_for_status()

Best practices for ML workloads

  • Use VPC-enabled domains to keep traffic private.
  • Design indices with appropriate sharding and lifecycle management (for example, hot-warm architectures for logs).
  • For vector search, store embeddings with k-NN index types and control dimensionality.
  • Use IAM and fine-grained access control to restrict index operations.
  • Consider downstream use (monitoring vs. search vs. RAG) when choosing index mappings.

Exam tips

  • OpenSearch appears in scenarios about log analytics, APM, or search over semi-structured data.
  • Vector search + RAG is a strong signal for OpenSearch or similar stores.
  • Firehose or CloudWatch Logs often feed data into OpenSearch in exam architectures.
