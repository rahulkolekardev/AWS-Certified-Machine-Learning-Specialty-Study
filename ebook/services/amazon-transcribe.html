Amazon Transcribe

Amazon Transcribe is an automatic speech recognition (ASR) service that converts speech to text. It supports batch and streaming transcription, custom vocabularies, and domain-specific features such as call analytics.

Key features

  • Batch transcription of audio files stored in S3.
  • Real-time streaming transcription over WebSocket or gRPC.
  • Custom vocabularies and language models to improve accuracy.
  • Speaker identification and channel identification for multi-party audio.
  • Call analytics for contact center use cases (Transcribe Call Analytics).

Where it fits in ML pipelines

  • Transcribing audio or video content to text for downstream NLP with Comprehend.
  • Creating datasets for voice-of-customer analysis, topic modeling, or sentiment tracking.
  • Enabling voice-based interfaces and search over recorded calls and meetings.
  • Feeding transcripts into labeling pipelines or training data for custom models.

How it works (high level)

CLI and SDK examples

Start a transcription job (CLI)

aws transcribe start-transcription-job \
  --transcription-job-name call-123 \
  --language-code en-US \
  --media MediaFileUri=s3://ml-audio/call-123.wav \
  --output-bucket-name ml-transcripts

Start streaming transcription (conceptual, Python)

# In practice, use the AWS SDK streaming APIs or WebSocket client.
# Here we show only the high-level idea.
import boto3

transcribe = boto3.client("transcribe")
# See official docs for full streaming examples.

Best practices for ML workloads

  • Choose the right language and domain settings to optimize accuracy.
  • Use S3 lifecycle policies to manage storage costs for large volumes of audio and transcripts.
  • Chain Transcribe with Comprehend for richer insights (sentiment, entities) on spoken content.
  • Use streaming transcription for real-time applications, batch for offline analytics.
  • Encrypt audio files and transcripts, and restrict access with IAM and S3 policies.

Exam tips

  • Transcribe is always the “speech to text” service; do not confuse it with Polly (text to speech).
  • Look for keywords like “call center,” “meeting recordings,” or “real-time captions.”
  • Transcribe plus Comprehend is a common pairing in exam architectures for voice-of-customer analytics.
