Amazon Simple Storage Service (S3)

Amazon S3 is a highly durable, scalable object storage service and the foundation of most AWS data lakes. For ML, it acts as the primary landing zone for raw data, feature datasets, model artifacts, and batch inference outputs, integrating with virtually every analytics and ML service on AWS.

Key features

  • Eleven 9s of durability with multiple storage classes for cost optimization.
  • Bucket policies, ACLs, and IAM for fine-grained access control.
  • Versioning, lifecycle policies, and replication for governance and resilience.
  • S3 Select and Glacier for selective and archival access.
  • Event notifications for downstream processing via Lambda, SNS, or SQS.

Where it fits in ML pipelines

  • Central data lake storage for raw, curated, and feature datasets.
  • Training data source and model artifact storage for SageMaker and custom training.
  • Intermediate storage for ETL outputs from Glue, EMR, and Flink.
  • Sink for batch inference outputs consumed by downstream systems or BI tools.

How it works (high level)

CLI and SDK examples

Create a bucket and upload data (CLI)

aws s3 mb s3://ml-data-lake-example
aws s3 cp local-data/ s3://ml-data-lake-example/raw/ --recursive

Download training data (Python, boto3)

import boto3

s3 = boto3.client("s3")
bucket = "ml-data-lake-example"

s3.download_file(bucket, "curated/train.parquet", "/tmp/train.parquet")

Best practices for ML workloads

  • Use consistent folder-style prefixes (for example, raw/, curated/, features/, models/) for organization.
  • Store analytics data in columnar formats (Parquet/ORC) partitioned by date and other keys.
  • Enable versioning and lifecycle policies to manage history and costs.
  • Use S3 server-side encryption (SSE) and VPC endpoints for private access from ML services.
  • Leverage S3 object tagging and Lake Formation for governance across many datasets.

Exam tips

  • S3 is almost always the default choice for ML data lakes and model artifact storage.
  • Remember partitioning and Parquet when questions mention Athena, Redshift Spectrum, or performance.
  • Private access from SageMaker or Bedrock typically involves S3 VPC endpoints.
