Amazon EMR

Amazon EMR is a managed big data service for running open-source frameworks such as Apache Spark, Hadoop, Hive, and Presto at scale. For ML, EMR is often used for large-scale data preparation, feature engineering, and distributed training with Spark MLlib or other libraries.

Key features

  • Managed clusters on EC2, EKS, or serverless (EMR Serverless) compute options.
  • Support for a wide range of big data engines including Spark, Hive, HBase, and Presto.
  • Deep integration with S3 for data lakes using EMRFS.
  • Autoscaling, spot instance integration, and instance fleets for cost optimization.
  • Notebook environments and integration with SageMaker Studio.

Where it fits in ML pipelines

  • Large-scale feature engineering on historical datasets stored in S3.
  • Complex joins and transformations that exceed single-node capacity.
  • Running distributed training jobs with Spark MLlib or custom distributed frameworks.
  • Preprocessing data for downstream consumption by SageMaker or Redshift.

How it works (high level)

CLI and SDK examples

Submit a Spark step (CLI)

aws emr add-steps \
  --cluster-id j-XXXXXXXXXXXXX \
  --steps Type=Spark,Name=ml-feature-job,Args=[--deploy-mode,cluster,--class,com.example.FeatureJob,s3://code-bucket/jar.jar]

List clusters (Python, boto3)

import boto3

emr = boto3.client("emr")

clusters = emr.list_clusters(ClusterStates=["RUNNING", "WAITING"])
for c in clusters["Clusters"]:
    print(c["Id"], c["Name"], c["Status"]["State"])

Best practices for ML workloads

  • Prefer EMR Serverless or EMR on EKS for simplified operations when you do not need long-lived clusters.
  • Store data in S3 and treat EMR clusters as ephemeral compute attached to the lake.
  • Use instance fleets and Spot Instances to reduce training and ETL costs.
  • Monitor with CloudWatch and configure automatic scaling based on YARN/Spark metrics.
  • Secure clusters using IAM roles, Kerberos (if needed), and private subnets in VPCs.

Exam tips

  • EMR vs. Glue: EMR offers more control and engine choice; Glue is fully serverless and opinionated.
  • Look for use cases mentioning Hadoop or specific engines like Spark, Hive, or HBase.
  • For petabyte-scale historical processing, EMR is often highlighted over per-query tools like Athena.
