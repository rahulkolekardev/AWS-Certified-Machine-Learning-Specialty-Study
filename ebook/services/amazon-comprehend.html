Amazon Comprehend

Amazon Comprehend is a natural language processing (NLP) service that uses machine learning to find insights and relationships in text. It offers pre-trained APIs and custom classification and entity recognition models without requiring deep NLP expertise.

Key features

  • Pretrained APIs for sentiment analysis, key phrases, entities, language detection, and syntax.
  • Custom classification and custom entity recognition using labeled datasets.
  • Support for batch and real-time processing.
  • Comprehend Medical for healthcare-specific entities (separate service).
  • Integration with S3, Kinesis, and other AWS services.

Where it fits in ML pipelines

  • Extracting text insights from customer feedback, reviews, and support tickets.
  • Creating input features such as sentiment scores or entity counts for downstream models.
  • Quickly prototyping NLP capabilities without building models from scratch.
  • Building domain-specific classifiers via custom classification APIs.

How it works (high level)

CLI and SDK examples

Detect sentiment (CLI)

aws comprehend detect-sentiment \
  --text \"The model performance exceeded our expectations.\" \
  --language-code en

Detect entities (Python, boto3)

import boto3

comprehend = boto3.client("comprehend")

text = "Alice bought a product from ExampleCorp in Seattle."
resp = comprehend.detect_entities(Text=text, LanguageCode="en")
for ent in resp["Entities"]:
    print(ent["Text"], ent["Type"], ent["Score"])

Best practices for ML workloads

  • Use Comprehend when you need standard NLP capabilities quickly and with minimal ML effort.
  • For domain-specific tasks, train custom models using labeled examples stored in S3.
  • Consider cost and latency when using Comprehend in synchronous paths; batch APIs can be cheaper.
  • For advanced or highly custom NLP, SageMaker or Bedrock may be more appropriate.
  • Secure text data using encryption at rest/in transit and appropriate IAM roles.

Exam tips

  • When you see “sentiment analysis” or “entity extraction” as a requirement, Comprehend is often the exam’s intended service.
  • Custom classification and custom entity recognition are important capabilities to remember.
  • Comprehend vs. Bedrock: Comprehend is task-specific; Bedrock provides general-purpose FMs.
