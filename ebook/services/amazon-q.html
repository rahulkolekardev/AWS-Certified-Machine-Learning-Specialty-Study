Amazon Q

Amazon Q is a family of generative AI–powered assistants that help developers, business users, and cloud practitioners work more efficiently. It provides specialized experiences, such as Q Developer for coding and Q Business for querying organizational data and content.

Key features

  • Q Developer: code generation, debugging help, and cloud troubleshooting assistance.
  • Q Business: semantic search and conversational access to enterprise knowledge bases.
  • Tight integration with AWS services, documentation, and consoles.
  • Enterprise controls for data sources, access policies, and auditability.
  • Powered by foundation models running on AWS (often via Bedrock under the hood).

Where it fits in ML pipelines

  • Assisting ML engineers and data scientists with code, infrastructure, and design questions.
  • Helping teams discover relevant internal documentation and datasets.
  • Accelerating cloud troubleshooting and optimization tasks for ML workloads.
  • Providing business stakeholders conversational insights based on analytics and BI outputs.

How it works (high level)

CLI and SDK examples

Amazon Q is primarily accessed via the AWS Console and integrations rather than a standalone public API.

Best practices for ML workloads

  • On the exam, see Q as a tool that accelerates teams building and operating ML solutions.
  • Configure access controls and data sources carefully to avoid exposing sensitive information.
  • Use Q to surface operational insights (for example, deployments, logs) to non-expert stakeholders.
  • Combine Q with Bedrock-based custom applications when you need bespoke user experiences.

Exam tips

  • Amazon Q is not a general-purpose model hosting platform; it is an AI assistant product.
  • For building your own generative apps, Bedrock or SageMaker is more appropriate.
  • Look for questions about assisting developers or business users rather than hosting models.
