AWS IoT Greengrass

AWS IoT Greengrass is an open-source edge runtime and cloud service for building, deploying, and managing applications on IoT devices. For ML, it enables running inference and data processing close to where data is generated, even with intermittent connectivity.

Key features

  • Local message bus and Lambda function execution on devices.
  • Support for containerized applications and native code.
  • Secure communication with AWS IoT Core and other services.
  • Over-the-air (OTA) deployments and configuration updates.
  • Integration with SageMaker for deploying edge models.

Where it fits in ML pipelines

  • Running ML inference locally on gateways or embedded devices.
  • Preprocessing sensor data before sending aggregated results to the cloud.
  • Supporting offline ML applications that sync periodically.
  • Deploying updated models from SageMaker or other registries to fleets of edge devices.

CLI and SDK examples

Greengrass deployments are typically managed via the console, CloudFormation, or the AWS IoT APIs. For full examples, see the official Greengrass documentation and SDKs.

Best practices for ML workloads

  • Design models and pipelines to fit the resource constraints of edge devices (memory, CPU, storage).
  • Use secure storage and communication for model binaries and inference data.
  • Plan for graceful degradation when connectivity to AWS is limited.
  • Monitor device health and inference metrics, forwarding summaries to CloudWatch or OpenSearch via IoT.

Exam tips

  • Greengrass appears when you need local ML inference or preprocessing at the edge.
  • Look for scenarios describing “intermittent connectivity” or “latency-sensitive edge control loops.”
  • SageMaker + Greengrass is a canonical pattern for edge model deployment.
