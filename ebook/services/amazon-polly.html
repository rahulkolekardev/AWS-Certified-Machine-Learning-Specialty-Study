Amazon Polly

Amazon Polly is a text-to-speech (TTS) service that turns text into lifelike speech. It provides multiple languages and voices, including neural TTS, and is commonly used for voice interfaces and content accessibility.

Key features

  • Standard and neural voices across many languages and locales.
  • Support for SSML to control pronunciation, emphasis, and pacing.
  • Real-time streaming and offline audio file generation.
  • Lexicon support to adjust pronunciation of domain-specific terms.
  • Integration patterns with Amazon Connect, web, and mobile apps.

Where it fits in ML pipelines

  • Providing spoken output for chatbots, virtual assistants, and accessibility features.
  • Generating audio content (for example, news reading or training materials) from text datasets.
  • Creating synthetic speech for testing voice-enabled ML applications.

How it works (high level)

CLI and SDK examples

Synthesize speech (CLI)

aws polly synthesize-speech \
  --text \"Welcome to the ML Specialty exam.\" \
  --output-format mp3 \
  --voice-id Joanna \
  output.mp3

Synthesize speech (Python, boto3)

import boto3

polly = boto3.client("polly")

resp = polly.synthesize_speech(
    Text="Your training job has completed successfully.",
    OutputFormat="mp3",
    VoiceId="Joanna",
)
with open("message.mp3", "wb") as f:
    f.write(resp["AudioStream"].read())

Best practices for ML workloads

  • Use neural voices when audio quality is a priority and cost allows.
  • Store frequently used prompts as audio files to reduce repeated synthesis costs.
  • Use SSML and lexicons to control pronunciation of technical terms or brand names.
  • Securely handle generated audio if it contains sensitive content.

Exam tips

  • Polly is strictly “text to speech”; Transcribe is “speech to text.”
  • Accessibility and voice interface scenarios typically call for Polly.
  • For IVR and contact center flows, Polly often appears with Amazon Connect.
