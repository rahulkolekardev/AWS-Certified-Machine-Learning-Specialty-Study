Amazon Bedrock

Amazon Bedrock is a fully managed service for building generative AI applications using foundation models from Amazon and leading model providers. It provides a unified API for text, image, and other generative tasks, along with tools for orchestration, guardrails, and retrieval-augmented generation.

Key features

  • Access to multiple foundation models (FMs) via a single API.
  • Model evaluation, prompt templates, and fine-tuning for select FMs.
  • Guardrails for controlling model behavior and filtering responses.
  • Knowledge bases and RAG support that connect models with enterprise data.
  • Agents that orchestrate multi-step workflows and tool use.

Where it fits in ML pipelines

  • Building chatbots, assistants, and content generation tools without managing GPU infrastructure.
  • Implementing RAG architectures that combine FMs with data stored in S3, OpenSearch, or other stores.
  • Rapid prototyping of generative experiences for internal and external users.
  • Complementing classical ML models with generative components.

How it works (high level)

CLI and SDK examples

Invoke a model (Python, boto3)

import boto3, json

bedrock = boto3.client("bedrock-runtime")

resp = bedrock.invoke_model(
    modelId="amazon.titan-text-lite-v1",
    body=json.dumps({"inputText": "Summarize this dataset description for ML."}),
    contentType="application/json",
    accept="application/json",
)
print(json.loads(resp["body"].read()))

Best practices for ML workloads

  • Use guarded endpoints and policies to control which applications can invoke which models.
  • Choose models based on latency, cost, and quality requirements; not all FMs are equal for every task.
  • For enterprise scenarios, connect Bedrock to governed data via knowledge bases and vector stores.
  • Monitor usage and responses with CloudWatch and CloudTrail for observability and compliance.
  • Combine Bedrock with traditional ML when tasks involve structured predictions or established models.

Exam tips

  • Bedrock is the preferred answer for managed generative AI on AWS with multiple FMs.
  • Questions mentioning “no GPU management,” “multiple foundation models,” or “RAG with knowledge bases” often point to Bedrock.
  • For custom, non-FM models, SageMaker remains the central service.
