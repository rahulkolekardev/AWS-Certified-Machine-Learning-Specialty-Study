Glossary

Data lake
Centralized storage, typically on S3, that holds raw and curated data in multiple formats for analytics and ML.
Feature
An individual input variable used by a model, often engineered from raw data (for example, “days since last purchase”).
Model training
The process of fitting model parameters to data, usually by minimizing a loss function using an optimization algorithm.
Inference
Using a trained model to generate predictions for new data, often served through an API endpoint.
Batch inference
Running predictions for large datasets offline, writing results to storage such as S3 or a data warehouse.
Online inference
Real-time prediction with low latency, typically used in user-facing applications.
Data drift
Change in the statistical properties of input data over time, potentially degrading model performance.
Concept drift
Change in the relationship between inputs and target labels, requiring retraining or new models.
Foundation model
A large pre-trained model (often generative) that can be adapted to many downstream tasks via prompting or fine-tuning.
Endpoint
A network-accessible URL or resource used to invoke a model for inference, such as a SageMaker or Bedrock endpoint.
VPC (Virtual Private Cloud)
Logically isolated section of the AWS cloud where you define networking, subnets, routing, and security groups.
IAM role
An AWS identity with attached permissions that can be assumed by users, services, or applications.
Managed service
A service where AWS operates the underlying infrastructure, reducing operational burden (for example, SageMaker, Bedrock).
Serverless
A compute model where you do not manage servers directly and pay primarily for actual usage (for example, Athena, Glue jobs).
Streaming data
Data that is continuously generated and delivered in small records, such as clickstreams or sensor readings.
Data catalog
Central metadata store describing datasets, schema, and locations. Glue Data Catalog and Lake Formation build on this concept.
Spot Instances
EC2 capacity at discounted prices with the risk of interruption, often used for cost-optimized training or batch jobs.
Blue/green deployment
Pattern where a new model version is deployed alongside the old one, with traffic gradually shifted after validation.
A/B testing
Experimentation approach that compares two or more variants (for example, models) by splitting live traffic and measuring outcomes.
Drift monitoring
Tracking input distributions and model quality metrics over time to detect when retraining is needed.
Explainability
Techniques such as SHAP values that help interpret why a model produced a given prediction.
Feature store
Central repository for storing, serving, and sharing ML features in online and offline modes.
GPU instance
An EC2 instance type with attached GPUs (for example, P- or G-family) used to accelerate training and inference.
ETL (Extract, Transform, Load)
Process of ingesting data, transforming it into usable form, and loading it into a target system for analytics or ML.
Lakehouse
An architecture that combines data lake flexibility with data warehouse-style management and performance.
Serverless inference
Model hosting where infrastructure is automatically scaled and billed based on requests rather than provisioned capacity.

Definitions aligned with AWS ML and data analytics terminology as used in official documentation. Last updated: 2025-11-16.
