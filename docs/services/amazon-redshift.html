Amazon Redshift

Amazon Redshift is a fully managed, petabyte-scale cloud data warehouse. For ML, it is used to store and query structured data, compute aggregates and features, and serve as the analytics layer feeding both BI tools and downstream ML pipelines.

Key features

  • Columnar storage and massively parallel processing for high-performance SQL analytics.
  • RA3 nodes and Redshift Serverless options for flexible scaling and pricing.
  • Redshift Spectrum for querying data in S3 without loading.
  • Integration with COPY and UNLOAD operations for data movement.
  • Built-in ML capabilities (Redshift ML) for training models using SQL with SageMaker under the hood.

Where it fits in ML pipelines

  • Storing curated, relational datasets used for analytics and feature engineering.
  • Computing aggregates, cohorts, and KPIs used as features for models.
  • Running Redshift ML to train simple models directly from SQL.
  • Serving as a high-performance source for feature extraction into SageMaker training jobs.

How it works (high level)

CLI and SDK examples

Create a Redshift Serverless namespace (CLI)

aws redshift-serverless create-namespace \
  --namespace-name ml-analytics

Run a query (Python, boto3)

import boto3

redshift_data = boto3.client("redshift-data")

resp = redshift_data.execute_statement(
    WorkgroupName="ml-workgroup",
    Database="ml_db",
    Sql="SELECT label, COUNT(*) FROM churn_events GROUP BY label",
)
print("Statement ID:", resp["Id"])

Best practices for ML workloads

  • Use RA3 or serverless with Redshift Spectrum to keep cold data in S3 and hot data in Redshift tables.
  • Design schemas with sort and distribution keys that match query patterns.
  • Use UNLOAD to export large training datasets to S3 in Parquet for SageMaker.
  • Consider Redshift ML for simple models but use SageMaker for complex or large-scale training.
  • Secure access with IAM, VPC, and encryption, especially for regulated analytics.

Exam tips

  • Redshift is the exam’s main data warehouse; Athena is serverless query on S3.
  • Redshift ML highlights integration with SageMaker for in-database model training.
  • RA3 + Spectrum is the key phrase for separating storage and compute.
