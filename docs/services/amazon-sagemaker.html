Amazon SageMaker

Amazon SageMaker is a fully managed machine learning service that provides tools to build, train, and deploy ML models at scale. It offers managed infrastructure, integrated development environments, MLOps workflows, and specialized features such as a feature store and model monitoring.

Key features

  • SageMaker Studio and notebooks for collaborative ML development.
  • Managed training jobs with automatic model tuning and spot instance support.
  • Real-time, batch, and serverless inference endpoints.
  • SageMaker Pipelines for CI/CD and MLOps automation.
  • Feature Store, Data Wrangler, Clarify (bias/explainability), and Model Monitor.

Where it fits in ML pipelines

  • End-to-end ML lifecycle: data preparation, training, evaluation, and deployment.
  • Hosting production endpoints for real-time or batch inference.
  • Managing feature definitions and reuse through SageMaker Feature Store.
  • Implementing reproducible pipelines and approvals via SageMaker Pipelines.

How it works (high level)

CLI and SDK examples

Launch a training job (CLI)

aws sagemaker create-training-job \
  --training-job-name ml-demo-job \
  --algorithm-specification TrainingImage=<ecr-image>,TrainingInputMode=File \
  --role-arn arn:aws:iam::123456789012:role/SageMakerRole \
  --input-data-config file://input-config.json \
  --output-data-config S3OutputPath=s3://ml-artifacts/train/ \
  --resource-config InstanceType=ml.m5.xlarge,InstanceCount=1,VolumeSizeInGB=50

Invoke an endpoint (Python, boto3)

import boto3, json

sm_runtime = boto3.client("sagemaker-runtime")

resp = sm_runtime.invoke_endpoint(
    EndpointName="ml-endpoint",
    ContentType="application/json",
    Body=json.dumps({"features": [0.1, 0.2, 0.3]}),
)
print(json.loads(resp["Body"].read()))

Best practices for ML workloads

  • Use managed training with Spot Instances for cost-efficient large jobs.
  • Separate staging and production endpoints and roll out changes via blue/green or canary deployments.
  • Use Pipelines for reproducible, auditable ML workflows and approvals.
  • Store features in Feature Store for consistency between offline training and online inference.
  • Monitor models with Model Monitor for drift and data quality issues.

Exam tips

  • SageMaker is the default answer for managed training and deployment when you want to avoid undifferentiated heavy lifting.
  • EC2/Batch are selected when you need bespoke control or nonstandard frameworks and are willing to manage infra.
  • Know key components: Studio, training jobs, endpoints, Pipelines, Feature Store, and Model Monitor.
