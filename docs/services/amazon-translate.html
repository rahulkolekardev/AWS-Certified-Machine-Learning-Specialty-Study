Amazon Translate

Amazon Translate is a neural machine translation service that delivers fast, high-quality language translation. It supports many language pairs and allows customization via terminology and parallel data.

Key features

  • Real-time and batch translation APIs.
  • Custom terminology to preserve domain-specific terms and brand names.
  • Active Custom Translation using parallel data for domain adaptation.
  • Integration with S3, CloudWatch, and other AWS services.
  • Support for a wide range of languages and scripts.

Where it fits in ML pipelines

  • Localizing user interfaces, content, and documentation.
  • Translating text data before downstream NLP (for example, sentiment analysis) when language support is limited.
  • Supporting multilingual chatbots and Q&A systems.
  • Expanding labeled datasets across languages by translating text.

How it works (high level)

CLI and SDK examples

Translate text (CLI)

aws translate translate-text \
  --text \"Hello, world\" \
  --source-language-code en \
  --target-language-code es

Translate text (Python, boto3)

import boto3

translate = boto3.client("translate")

resp = translate.translate_text(
    Text="Machine learning is powerful.",
    SourceLanguageCode="en",
    TargetLanguageCode="fr",
)
print(resp["TranslatedText"])

Best practices for ML workloads

  • Use custom terminology to preserve product names and technical jargon.
  • For high-volume translation, use batch APIs with S3 integration.
  • When combining with Comprehend, keep track of original language and translation quality assumptions.
  • Be aware of cultural context when interpreting translated text in ML models.

Exam tips

  • Translate is the dedicated service for language translation, not for general text generation.
  • Look for phrases about “localizing content” or “supporting multiple languages.”
  • Custom terminology is often highlighted in questions about preserving brand or product names.
