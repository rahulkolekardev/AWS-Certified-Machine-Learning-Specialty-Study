Amazon Textract

Amazon Textract is a document analysis service that uses machine learning to extract text, forms, tables, and key-value pairs from scanned documents. It goes beyond traditional OCR by understanding structure and relationships in documents.

Key features

  • Extraction of text, forms, and tables from images and PDFs.
  • Key-value pair detection to identify field names and values.
  • Queries feature for asking targeted questions of documents (for example, “Total due”).
  • Support for asynchronous batch processing via S3.
  • Integration with Comprehend for downstream NLP on extracted text.

Where it fits in ML pipelines

  • Digitizing paper-based workflows such as invoices, forms, and statements.
  • Extracting structured data for downstream analytics and ML models.
  • Creating labeled datasets for document understanding tasks.
  • Feeding extracted fields into fraud detection or risk scoring models.

How it works (high level)

CLI and SDK examples

Detect document text (CLI)

aws textract detect-document-text \
  --document \"S3Object={Bucket=ml-docs,Name=invoice.pdf}\"

Analyze document (Python, boto3)

import boto3

textract = boto3.client("textract")

resp = textract.analyze_document(
    Document={"S3Object": {"Bucket": "ml-docs", "Name": "form.png"}},
    FeatureTypes=["FORMS", "TABLES"],
)
for block in resp["Blocks"]:
    if block["BlockType"] == "KEY_VALUE_SET":
        print(block.get("Text", ""))

Best practices for ML workloads

  • Use high-resolution scans and appropriate formats (for example, 300 DPI) for best accuracy.
  • For large document sets, use asynchronous APIs and process results from S3.
  • Normalize and validate extracted fields before using them as ML features.
  • Combine Textract with Comprehend for deeper semantic understanding of extracted text.

Exam tips

  • Textract is for document text, forms, and tables; Rekognition is for general images and scenes.
  • Look for keywords like “scanned PDFs,” “forms,” “table extraction,” or “key-value pairs.”
  • Textract plus Comprehend is a common pattern for end-to-end document understanding.
