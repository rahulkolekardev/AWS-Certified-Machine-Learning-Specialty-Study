Amazon Elastic Compute Cloud (EC2)

Amazon EC2 provides resizable virtual machines in the cloud. For ML, EC2 offers CPU and GPU instances that you can configure with any frameworks or runtimes, giving maximum flexibility at the cost of more operational responsibility than fully managed ML services.

Key features

  • Wide range of instance types, including GPU families for training and inference.
  • On-Demand, Reserved, and Spot pricing models.
  • Auto Scaling groups, placement groups, and elastic block storage (EBS).
  • Deep integration with VPC, IAM, and other core AWS services.
  • Ability to install any ML framework, library, or runtime you need.

Where it fits in ML pipelines

  • Custom training or inference setups that do not fit nicely into SageMaker or Bedrock.
  • Legacy or specialized ML workloads that require custom OS or drivers.
  • Running open-source tools and orchestrators (for example, Kubeflow, Airflow) self-managed.
  • Prototyping and experimentation with full root access to the environment.

How it works (high level)

CLI and SDK examples

Launch an instance (CLI)

aws ec2 run-instances \
  --image-id ami-0123456789abcdef0 \
  --instance-type g5.xlarge \
  --count 1 \
  --iam-instance-profile Name=MLInstanceProfile \
  --security-group-ids sg-1234567890abcdef0 \
  --subnet-id subnet-1234567890abcdef0

Describe instances (Python, boto3)

import boto3

ec2 = boto3.client("ec2")
resp = ec2.describe_instances(
    Filters=[{"Name": "instance-state-name", "Values": ["running"]}]
)
for r in resp["Reservations"]:
    for inst in r["Instances"]:
        print(inst["InstanceId"], inst["InstanceType"])

Best practices for ML workloads

  • Use appropriate GPU instances (P, G, or newer families) for deep learning workloads.
  • Automate provisioning via infrastructure-as-code (CloudFormation, CDK, Terraform) to avoid manual drift.
  • Use Auto Scaling and Spot Instances for cost-effective batch training jobs.
  • Place ML instances in private subnets with controlled access via load balancers and security groups.
  • Prefer SageMaker for managed training and hosting unless you have strong reasons to manage EC2 directly.

Exam tips

  • EC2 is chosen when you need full control or unsupported frameworks, not when you want managed ML.
  • Cost optimization questions often mention Spot Instances and right-sizing instance types.
  • Network and security details (VPC, security groups, IAM roles) frequently appear in EC2-based ML scenarios.
