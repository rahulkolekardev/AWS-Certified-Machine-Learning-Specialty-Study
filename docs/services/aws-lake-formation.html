AWS Lake Formation

AWS Lake Formation helps you build, secure, and manage data lakes on AWS. It provides fine-grained access control over data stored in S3 and registered in the Glue Data Catalog, integrating with analytics and ML services that query the lake.

Key features

  • Central permissions model for databases, tables, and columns in the data lake.
  • Integration with AWS Glue Data Catalog, Athena, Redshift Spectrum, and EMR.
  • Support for row-level and column-level security policies.
  • Blueprints and workflows for ingesting data into governed tables.
  • Tag-based access control for simplifying permissions management.

Where it fits in ML pipelines

  • Governed data lakes where multiple teams share curated datasets for ML and analytics.
  • Scenarios requiring sensitive attributes (for example, PII) to be restricted to certain roles.
  • Managing cross-account and cross-team access to feature tables stored in S3.
  • Auditable permission model for regulated industries building ML solutions.

How it works (high level)

CLI and SDK examples

Grant table permissions (CLI)

aws lakeformation grant-permissions \
  --principal DataLakePrincipalIdentifier=arn:aws:iam::123456789012:role/AnalystRole \
  --resource '{\"Table\":{\"DatabaseName\":\"ml_lake\",\"Name\":\"features_customers\"}}' \
  --permissions SELECT

List permissions (Python, boto3)

import boto3

lf = boto3.client("lakeformation")

resp = lf.list_permissions(
    ResourceType="TABLE",
    MaxResults=20,
)
for entry in resp.get("PrincipalResourcePermissions", []):
    print(entry["Principal"], entry["Resource"], entry["Permissions"])

Best practices for ML workloads

  • Use Lake Formation for centralized permissions instead of duplicating S3 bucket policies per service.
  • Leverage LF-tags and tag-based access control to simplify large-scale permission management.
  • Apply column-level permissions for sensitive attributes, allowing broader access to non-sensitive fields.
  • Integrate Lake Formation with Athena, Redshift, and EMR to enforce consistent governance.
  • Design your data model (databases, tables, columns) with permission boundaries in mind.

Exam tips

  • Questions about fine-grained (table/column) access to S3-based lake data usually point to Lake Formation.
  • Remember that Lake Formation uses the Glue Data Catalog underneath.
  • Contrast with IAM-only controls; Lake Formation provides higher-level, data-centric permissions.
